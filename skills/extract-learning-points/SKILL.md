# 提取学习要点

阅读用户提供的材料，并产出面向学习场景的结构化结果。

聚焦于提取材料中的主要信息。除非用户明确要求，否则不要展开解释 harness 架构、agent loop 或仓库内部实现。

## 工作流程

1. 识别材料来源及其局限。
2. 阅读足够的内容，确认主题、结构和信息密度。
3. 区分直接事实与解释性推断。
4. 提取能够保留核心含义的最小信息集合。
5. 输出一个便于学习、浏览和复用的结构化结果。

## 输出结构

除非用户要求其他格式，否则使用以下结构：

```markdown
主题：<一句话>

摘要：
<2-4 句>

关键点：
- <要点 1>
- <要点 2>
- <要点 3>

重要术语：
- <术语>：<基于材料的简短解释>

开放问题：
- <由信息缺口、歧义或后续值得核查的地方引出的提问>

后续建议：
- <接下来可继续阅读、比较、练习或验证的内容>
```

如果材料很短，就减少条目数量，不要为了凑格式而填充空泛内容。

如果材料技术性强、信息密度高，就缩短“摘要”，把重点放在“关键点”和“重要术语”上。

## 提取规则

先说明材料明确表达了什么，再说明你根据材料推断了什么。

保留作者的主要论点、定义、机制、对比关系和结论。

积极压缩重复内容。只保留那些会影响理解、支撑结论、定义概念或影响后续学习的点。

优先使用具体陈述，不要使用空泛概括。不要写“文本讨论了若干观点”，而要直接写出那些观点。

如果材料包含过程或步骤，要把顺序表达清楚。

如果材料包含对比，要保留对比维度。

如果材料包含不确定性或权衡，要明确保留下来。

## 来源处理

对于 `.md` 和 `.txt`：

- 把标题、列表和强调视为弱结构提示。
- 利用章节边界组织概念分组。

对于 `.pdf` 或从 `.pdf` 提取出的文本：

- 预期会出现断行、页眉页脚和阅读顺序噪声。
- 先恢复句子和段落意图，再提取要点。
- 对重复出现的页面装饰信息默认忽略，除非它会改变语义。

对于质量混杂的输入：

- 不要用想当然的方式补平缺失上下文，要显式报告不确定性。
- 当某个结论只是“看起来成立”但没有被当前材料充分支持时，要明确指出。

## 不确定性规则

不要编造缺失的事实、来源、例子或定义。

当信息不完整时，可以明确使用类似表述：

- `材料没有明确说明……`
- `这看起来意味着……，但原文没有直接这样表述。`
- `材料在……这一点上存在歧义。`

“开放问题”必须来自真实信息缺口，而不是泛泛而谈的好奇心提问。

## 优先级启发式

按以下顺序确定提取重点：

1. 中心主题或核心论点
2. 定义与核心概念
3. 机制、论证过程或因果关系
4. 重要例子或证据
5. 限制、条件与未解决问题

删除装饰性细节、修辞性填充和重复示例，除非这种重复本身具有意义。

## 适配指引

如果用户是为了备考，优先提取定义、区分点和可能被考到的内容。

如果用户要快速简报，优先给出“摘要”和最重要的 3-5 个“关键点”。

如果用户要深入学习笔记，保持同一结构，但扩展“重要术语”和“开放问题”。

如果用户提供了多份文档，先分别提取每份文档的主题，再综合归纳它们的共同点与差异。
